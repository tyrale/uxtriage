Stack Rank Prioritization Consultation

Whats the process to get it built?

The need to prioritize comes from a very simple fact: we just don’t have enough resources to work on everything we can come up with.

Thus, we need a process to determine the sets (and sequence) of things that should be done on the product to deliver the most value at each point in time, given our constraints.

If we break this statement down, a core group of questions then need to be answered:

  1. How can we know what’s valuable? How valuable is it? Valuable to whom?
  2. How can we define the set of things that should go together in a product release? How should we sequence those releases?
  3. How can we get the necessary buy-in to follow through and get these things to the market?
  4. How can we know if our assumptions are right? Are we on the right track? Are we really delivering value? Could we do any better?

With UX Triage you'll walk away with a workable timeline, and a process for updating as you go.
